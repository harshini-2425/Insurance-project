FULL-SCREEN DOCUMENT VIEWER FIX - COMPLETE
==========================================

PROBLEM FIXED:
When viewing PDF or documents in the admin document viewer, the document was displayed 
in a constrained modal with the PDF appearing very small and zoomed out, causing the 
header (company logo, title) to be cropped and unreadable.

ROOT CAUSES:
1. Modal container used maxWidth: 90vw and maxHeight: 95vh (limiting screen usage)
2. Centered modal layout with padding wasted space
3. PDF iframe had minHeight constraints but no full height
4. Footer taking bottom space
5. No full-screen experience: just a centered box

FIXES IMPLEMENTED:
==================

1. FULL-SCREEN LAYOUT (DocumentPreviewModal.jsx lines 147-165)
   OLD: Centered modal with maxWidth: 90vw, maxHeight: 95vh
   NEW: Full-screen overlay with width: 100%, height: 100vh
   
   Result: Document uses entire viewport

2. HEADER BAR (lines 166-195)
   CHANGES:
   - Reduced padding to 12px (was 16px)
   - Added flexShrink: 0 to preserve header size
   - Optimized title/document type display
   - Close button always accessible
   
   Result: Minimal overhead, maximum content space

3. CONTENT AREA (lines 197-216)
   CHANGES:
   - Use flex: 1 to fill all remaining space
   - height: calc(100vh - 60px) for precise sizing
   - Direct children fill 100% width/height
   - Removed unnecessary padding and wrappers
   
   Result: Document expands to use all available space

4. PDF IFRAME (lines 325-334)
   OLD: width: 100%, height: 100%, border: none, minHeight: 600px, flex: 1
   NEW: width: 100%, height: 100%, border: none, display: block
   
   Result: PDF fills entire content area with proper zoom

5. IMAGE VIEWER (lines 336-356)
   CHANGES:
   - Full width/height container
   - Image maxWidth/Height: 95% (centered with margin)
   - backgroundColor: #e5e7eb for contrast
   
   Result: Images displayed properly centered at maximum usable size

6. TEXT FILE VIEWER (lines 358-368)
   CHANGES:
   - Direct iframe, no wrapper
   - width: 100%, height: 100%
   - Removed padding and flex constraints
   
   Result: Text files display at full size

7. FOOTER REMOVAL (previously lines 363-402)
   CHANGE: Removed entire footer bar
   REASON: Close button in header, full screen doesn't need footer
   
   Result: More space for document, cleaner interface

8. ESC KEY SUPPORT (lines 35-50)
   ADDED: useEffect hook to handle ESC key press
   ADDED: document.body.style.overflow = 'hidden' when open
   ADDED: Cleanup event listeners on close
   
   Result: Professional fullscreen experience with ESC to close
