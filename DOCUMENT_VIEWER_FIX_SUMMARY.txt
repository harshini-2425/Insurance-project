DOCUMENT VIEWER LAYOUT FIX - COMPLETE
=====================================

ISSUE FIXED:
When viewing uploaded documents (PDF/DOC) in the View Document page, the document 
header was only partially visible due to incorrect iframe/container sizing.

ROOT CAUSES IDENTIFIED:
1. Modal maxHeight: 90vh - too small and restrictive
2. Modal maxWidth: 56rem (fixed width) - not responsive
3. Content area padding: 16px - wasted space needed for document viewing
4. PDF iframe minHeight: 500px - insufficient for larger documents
5. Header/Footer had no flexShrink - could collapse the content area
6. Iframes missing flex: 1 - didn't expand to fill available space
7. Missing minHeight: 0 on content - flex layout issues

FIXES APPLIED TO DocumentPreviewModal.jsx:
==========================================

1. MODAL CONTAINER (lines 151-160)
   BEFORE: maxWidth: '56rem', maxHeight: '90vh'
   AFTER:  maxWidth: '90vw', maxHeight: '95vh'
   
   ✅ Responsive width (90% viewport)
   ✅ Larger height (95% viewport)
   ✅ Full document visibility

   ✅ Prevents header from collapsing
