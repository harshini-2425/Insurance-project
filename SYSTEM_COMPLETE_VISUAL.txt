API Endpoints
  ✅ POST /api/admin/claims/{claim_id}/approve
     - Requires: token, is_admin=true
     - Updates: claim status → "approved"
     - Creates: notification for user
     - Logs: audit trail entry
     - Response: {success, claim, notification}

  ✅ POST /api/admin/claims/{claim_id}/reject
     - Requires: token, is_admin=true, reason (MANDATORY)
     - Updates: claim status → "rejected"
     - Creates: notification with reason
     - Logs: audit trail with reason
     - Response: {success, claim, notification}

  ✅ GET /api/admin/dashboard/stats
     - Requires: token, is_admin=true
     - Returns: {total_claims, pending_claims, approved_count, rejected_count}
     - Cached: Updates every poll interval
     - Performance: <50ms query time

  ✅ GET /api/admin/audit-logs
     - Requires: token, is_admin=true
     - Returns: List of admin actions with timestamps
     - Filterable: by admin_id, action_type, date range
     - Paginated: limit, offset parameters

  ✅ GET /api/user/notifications
     - Requires: token
     - Returns: {notifications[...], unread_count}
     - Paginated: limit, offset
     - Ordered: by created_at DESC
     - User-Isolated: Only their notifications

  ✅ POST /api/notifications/{notification_id}/read
     - Requires: token
     - Updates: status → "read", read_at → NOW()
     - Response: {success, notification}

Admin Approval Workflow
  ┌─ Admin clicks "Approve" on claim
  │
  ├─ Frontend posts to /api/admin/claims/{id}/approve
  │
  ├─ Backend processes:
  │  ├─ Validates JWT token ✅
  │  ├─ Checks is_admin=true ✅
  │  ├─ Starts database transaction ✅
  │  ├─ Updates claim status → "approved" ✅
  │  ├─ Sets reviewed_at & reviewed_by ✅
  │  ├─ Creates ClaimNotification row ✅
  │  ├─ Creates AdminLog entry ✅
  │  └─ Commits transaction ✅
  │
  ├─ Frontend receives success response
  │
  ├─ Frontend updates state:
  │  ├─ Removes claim from pending list ✅
  │  ├─ Updates stats (pending -1, approved +1) ✅
  │  ├─ Shows success toast ✅
  │  └─ Refreshes audit log ✅
  │
  └─ User logs in or has tab open
     │
     ├─ NotificationBell polls every 3s
     │
     ├─ Sees new notification in dropdown:
     │  ├─ Title: "Claim #CLM-2024-001 Approved" ✅
     │  ├─ Message: "[claim details]" ✅
     │  ├─ Status: "unread" (blue dot) ✅
     │  └─ Time: "just now" ✅
     │
     ├─ MyClaims claim updates:
     │  ├─ Status badge changes to "✓ Approved" (green) ✅
     │  ├─ Admin comment displays ✅
     │  ├─ Timestamp shows update ✅
     │  └─ No page reload needed ✅
     │
     └─ User can click notification
        └─ Marks as read, blue dot disappears ✅

Admin Rejection Workflow
  ┌─ Admin clicks "Reject" button
  │
  ├─ Frontend requires rejection reason
  │  └─ Can't proceed without it ✅
  │
  ├─ Admin enters reason & confirms
  │
  ├─ Backend processes:
  │  ├─ Updates claim status → "rejected" ✅
  │  ├─ Stores rejection reason ✅
  │  ├─ Creates notification with reason ✅
  │  ├─ Logs admin action ✅
  │  └─ Commits transaction ✅
  │
  └─ User notification received:
     ├─ Title: "Claim #CLM-2024-001 Rejected" ✅
     ├─ Message includes rejection reason ✅
     ├─ Admin comment visible in claim ✅
     └─ Claim badge shows "✗ Rejected" (red) ✅
